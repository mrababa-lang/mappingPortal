# SlashData Vehicle Portal - Enhancement & Fixes Log

This document outlines the roadmap to transition the application from the current "Mock/Prototype" state to a robust "Production" system backed by Java Spring Boot and MySQL.

**Status Key:**
[X] = Implemented
[ ] = Pending

================================================================================
SECTION 1: BACKEND MIGRATION (Priority High)
================================================================================

## 1. Architecture & Routing
- [ ] **React Router Integration**: 
  - Replace state-based `currentView` in `App.tsx` with `react-router-dom`.
  - Implement Protected Routes for authentication checks.
  - Enable deep linking (e.g., directly navigating to `/adp/mapping`).

## 2. API Service Layer
- [ ] **Replace StorageService**:
  - Create `services/api.ts` using `axios`.
  - Replace local `localStorage` CRUD operations with REST API calls.
  - Implement Request/Response interceptors for JWT injection and global error handling.
- [ ] **ID Management**:
  - Transition from Frontend UUIDs (String) to Backend Auto-Increment IDs (Long/BigInt) for Makes, Models, and Users.

## 3. Security & AI
- [ ] **Server-Side AI Proxy**:
  - Move `geminiService.ts` logic to the backend to protect the API Key.
  - Frontend should call `POST /api/ai/suggest` instead of contacting Google directly.
- [ ] **Authentication**:
  - Replace mock login in `views/Login.tsx` with real JWT authentication (`POST /api/auth/login`).
